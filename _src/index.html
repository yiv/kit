🚀  在你的机构中引入Go语言

Go是一门很轻巧且非常适合开发微服务的语言。 Go语言填补了其它标准库留下的空白，可以让你的团队信心满满地把Go引入到你们的技术栈中。

🔍  聚集你的业务逻辑

采用微服务意味着构建一个分布式系统，这将引入很多挑战。 Go kit 在大多数常见的运维操作和基础设施方面提供指引和方案。 帮助你把精力聚集在你的业务。

😌  少量的建议，轻量的支持

你比任何人都更了解你的业务领域和环境. Go kit 是一套不带成见的工具, 从最开始的设计目标就是 互通有无 。 你可以选用最适合你的数据库、组件、平台和框架。

为什么选择 Go？

Go 语言设计的首要原则是提升软件工程实践。 简单、易掌握，— 最重要的是 — 便于大型和动态的软件团队维护。 因为具备高性能的并发、丰富的标准库和持续稳定提升的运行时，用来开发微服务它几近完美。

为什么选择微服务？

几乎同时代的所有软件工程师都在关注如何缩短软件发布周期。 微服务作为面向服务的框架设计模式，在得优雅地消除团队间摩擦方面是一种革命性的创举，可以给工程师和团队足够的自治，以便他们持续地交付、迭代和改进。

为什么选择 Go kit?

Go 是一门优秀的通用编程语言，但是微服务在 RPC 安全、系统可观察性、设施整合，甚至于软件设计等方面需要一些特殊的支持， Go kit 可以填补这些标准库所不具有的功能，让 Go 在任何开发微服务的组织当中成为首选编程语言。
